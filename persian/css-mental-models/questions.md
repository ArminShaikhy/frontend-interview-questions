### 1. چگونه مدل جعبه CSS را مفهوم‌سازی می‌کنید و تأثیر آن بر تصمیم‌گیری‌های چیدمان در یک برنامه React چیست؟

**مدل جعبه CSS** نحوه اندازه‌گیری و فاصله‌گذاری عناصر را تعریف می‌کند:

```css
Total size = content + padding + border + margin

در React، کامپوننت‌ها را به‌عنوان کانتینرهای چیدمان یا بلوک‌های محتوا در نظر می‌گیرم، و درک مدل جعبه برای موارد زیر حیاتی است:

یکنواختی فاصله‌گذاری (مثل جلوگیری از فروپاشی حاشیه)
اندازه‌گیری قابل پیش‌بینی با box-sizing: border-box
جلوگیری از اشکالات سرریز در ساختارهای تودرتو

مثال:
* {

}

این تضمین می‌کند که padding و border به‌طور غیرمنتظره روی عرض/ارتفاع تأثیر نگذارند.


2. مدل ذهنی شما برای مدیریت خاصیت و وراثت CSS در یک پروژه بزرگ React چیست؟
من از اصل اولویت خاصیت پایین پیروی می‌کنم:

استفاده از استایل‌های محدود به کامپوننت (CSS Modules یا CSS-in-JS) برای کاهش درگیری‌های خاصیت جهانی.
ترجیح انتخابگرهای کلاس تکی.
اجتناب از انتخابگرهای بیش‌ازحد مشخص (div.btn.primary) که بازنویسی را شکننده می‌کنند.

مثال با CSS Modules:
/* Button.module.css */
.button {
  color: white;
}

در React:
<button className={styles.button}>ارسال</button>

همچنین استفاده از !important را محدود می‌کنم و برای ویژگی‌های حیاتی مثل چیدمان از وراثت اجتناب کرده و به‌جای آن از اظهارات صریح استفاده می‌کنم.


3. رابطه بین مدل‌های چیدمان CSS (مثل Flexbox، Grid) و سلسله‌مراتب کامپوننت‌های React را چگونه در نظر می‌گیرید؟
من مسئولیت‌های چیدمان را با مرزهای کامپوننت هم‌راستا می‌کنم:

Flexbox: برای هم‌ترازی یک‌بعدی در داخل یک کامپوننت (مثل نوارهای ناوبری، کارت‌ها).
CSS Grid: برای چیدمان دوبعدی در سطوح بالاتر (مثل پوسته‌های صفحه، داشبوردها).

درخت کامپوننت React ↔ مدل چیدمان CSS:

کامپوننت کانتینر = سیستم چیدمان (Grid/Flex)
کامپوننت‌های برگ = بلوک‌های محتوا (موقعیت‌یابی در چیدمان والد)

مثال:
<GridLayout>
  <Sidebar />
  <MainContent />
</GridLayout>

این رویکرد جداسازی منطق چیدمان و محتوا را تشویق می‌کند.


4. رویکرد شما برای مدل‌سازی طراحی پاسخ‌گو در CSS برای یک برنامه React چیست؟
من CSS موبایل‌محور با توکن‌های ابزار و سیستم‌های طراحی را ترجیح می‌دهم.
رویکرد:

استفاده از کوئری‌های @media یا ابزارهای رسانه‌ای CSS-in-JS
ساختار نقاط شکست از طریق شیء تم یا پیکربندی
ترجیح واحدهای نسبی (rem، %) برای مقیاس‌پذیری

مثال (Styled Components):
const Card = styled.div`
  padding:  click here to chat with Grok 1rem;
  @media (min-width: 768px) {
    padding: 2rem;
  }
`;

در مقیاس بزرگ، از توکن‌های طراحی و یک سیستم چیدمان متمرکز برای حفظ یکنواختی استفاده می‌کنم.


5. چگونه درباره عملکرد CSS (مثل reflowها، repaintها) در زمینه رندر مجدد React فکر می‌کنید؟
اصل کلیدی: کاهش تکان‌های چیدمان.
من:

از تحریک مکرر ویژگی‌های پرهزینه CSS (مثل width، top) اجتناب می‌کنم.
برای انیمیشن‌ها transform و opacity را ترجیح می‌دهم—این‌ها reflow را تحریک نمی‌کنند.
هنگام تعامل با چیدمان (مثل مدیریت تغییر اندازه)، خواندن/نوشتن DOM را محدود می‌کنم.

در React:

از جهش‌های غیرضروری DOM در رندرها جلوگیری می‌کنم.
از memo، useCallback برای جلوگیری از تحریک reflowهای پرهزینه از طریق به‌روزرسانی‌های DOM استفاده می‌کنم.



6. مدل ذهنی شما برای محدود کردن CSS در یک برنامه React (مثل CSS Modules، Styled Components) چیست؟
هر کامپوننت را به‌عنوان یک مرز استایل در نظر می‌گیرم. محدودسازی CSS از نشت و بازنویسی جلوگیری می‌کند.
CSS Modules:

ساده، سریع، مبتنی بر کلاس.
عالی برای جریان‌های کاری سنتی CSS.

Styled Components / Emotion:

استایل‌های کاملاً محصور
هم‌مکانی استایل‌ها با منطق، استفاده از پراپ‌ها برای استایل‌دهی پویا.

مثال:
const Button = styled.button`
  background: ${({ primary }) => (primary ? "blue" : "gray")};
`;

برای برنامه‌های بزرگ، اغلب از CSS Modules + کلاس‌های ابزار (مثل Tailwind) برای تعادل استفاده می‌کنم.


7. چگونه یک معماری CSS برای پشتیبانی از تم‌سازی (مثل حالت تاریک) در یک برنامه React طراحی می‌کنید؟
تم‌ها را با استفاده از ویژگی‌های سفارشی CSS و یک زمینه/ارائه‌دهنده تم طراحی می‌کنم.
CSS:
:root {
  --bg-color: white;
}
[data-theme="dark"] {
  --bg-color: #121212;
}

React:
<body data-theme={theme}>...</body>

از یک سوئیچ تم برای به‌روزرسانی data-theme استفاده می‌کنم و حالت تم را در زمینه جهانی یا از طریق localStorage مدیریت می‌کنم.
جایگزین Styled Components:
<ThemeProvider theme={lightTheme}>...</ThemeProvider>

9. رویکرد شما برای مدل‌سازی انیمیشن‌ها و ترنزیشن‌های CSS در یک برنامه React چیست؟
انیمیشن‌ها را به دو دسته تقسیم می‌کنم:

ترنزیشن‌های رابط کاربری مبتنی بر حالت (مودال‌ها، منوهای کشویی) → استفاده از Framer Motion یا React Transition Group.

/>
